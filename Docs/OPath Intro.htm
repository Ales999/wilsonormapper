Introduction

The Object Path Language (OPath) is a query language developed by Microsoft which allows you to query for objects stored in relational databases using standard object-oriented syntax.  The OPath language is loosely based on XML Path Language (XPath) and contains elements familiar to both Visual Basic .NET and C# developers.  OPath also includes several operators for complex value comparisons and enables you to traverse object relationships like you would using standard application code.

Version 4.0 of Wilson ORMapper introduced an implementation of Microsoft's OPath language and supporting API based on the documentation found in the December CTP of Windows "Longhorn" SDK (Build 4074).  Microsoft was still developing this technology when this implementation was created but, as of October 2005, it appears Microsoft has abandoned that effort and directed its attention elsewhere; specifically to Linq and DLinq.  While this it an unfortunate turn of events, the technology still provides incredible value and additional updates and enhancements will continue to be made to the OPath implementation in the Wilson ORMapper.

What's New

Version 4.2 contains the following enhancements:

  • String functions RIGHT, LEFT, UPPER, LOWER have been added.  See OPath Functions for details.
  • Sort expressions can now contain relationship traversals, which allows sorting records using values from related tables.  For example, sorting a list of Products by Category Name where Products are assigned to a Category by ID.  See Sorting Query Results for details.
  • Support for the new Lookup field type (new in 4.2) has been added.  Lookup fields can be used in queries.
  • A few minor bugs were also corrected.
